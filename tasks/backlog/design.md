# Design Backlog

## Purpose

This backlog tracks UX and design-system work required for the MVP.

## MVP Design Epics

### 1. App Shell Design

### Tasks

- Design primary sidebar.
- Design top bar.
- Design repository switcher.
- Design active run indicator.
- Design pending approval indicator.
- Design secondary panel behavior.

### Acceptance Criteria

- Users can orient themselves quickly.
- Active workspace, repository, run, and approvals are visible.

### 2. Repository Overview

### Tasks

- Design repository summary layout.
- Design scan status states.
- Design stack and tooling summary.
- Design documentation summary.
- Design Git summary.
- Design validation candidate list.

### Acceptance Criteria

- Repository state is scannable and evidence-oriented.

### 3. Task And Plan Flows

### Tasks

- Design task editor.
- Design requirements and acceptance criteria sections.
- Design plan review surface.
- Design assumptions, risks, and validation plan display.
- Design plan approval flow.

### Acceptance Criteria

- User can move from task to approved plan without losing context.

### 4. Agent Run Experience

### Tasks

- Design run timeline.
- Design context source list.
- Design tool call events.
- Design pause and approval states.
- Design final output summary.

### Acceptance Criteria

- Agent work is inspectable and calm.

### 5. Change Review

### Tasks

- Design changed file list.
- Design diff review shell.
- Design risk summary.
- Design validation summary.
- Design accept, revise, reject actions.

### Acceptance Criteria

- User can review AI-generated work confidently.

### 6. Design System Foundations

### Tasks

- Define initial color token values.
- Define initial typography token values.
- Define spacing and layout primitives.
- Define button and icon button variants.
- Define badge/status components.
- Define dialog and approval components.

### Acceptance Criteria

- MVP screens can be built with consistent primitives.

### 7. Accessibility Review

### Tasks

- Define focus states.
- Define keyboard navigation expectations.
- Define status non-color indicators.
- Define tooltip and truncation behavior.
- Review approval and command palette accessibility.

### Acceptance Criteria

- MVP interaction surfaces meet baseline accessibility expectations.
